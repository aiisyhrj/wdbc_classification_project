1. WHICH CLASSIFIERS DID YOU TEST?
------------------------------------------------------------
Four classifiers were tested on the Breast Cancer Wisconsin dataset,
using an 80/20 train/test split (random_state=42):

  - SVM (Support Vector Machine): finds the optimal separating
    hyperplane with maximum margin between classes.

  - Decision Tree: recursively splits data by the feature that
    gives the best information gain at each node.

  - KNN (K-Nearest Neighbours): classifies each sample by
    majority vote among its k closest training neighbours.

  - Random Forest: an ensemble of many decision trees that vote
    together; reduces overfitting compared to a single tree.

5. WHY IS DOCUMENTATION IMPORTANT IN AN OPEN-SOURCE ML PROJECT?
------------------------------------------------------------
Good documentation is critical for several reasons:

  a. Reproducibility — other developers can re-run your experiment
     and get the same results by following the README and
     requirements.txt.

  b. Transparency — inline comments and a clear report explain
     WHY decisions were made (e.g. why we use an 80/20 split,
     which metric to trust), not just what the code does.

  c. Collaboration — a well-structured project lets teammates or
     contributors understand and extend the code without having
     to decode it from scratch.

  d. Trust — in medical ML applications, doctors and regulators
     need to understand how a model works before trusting its
     predictions. Documentation bridges the gap between engineers
     and non-engineers.

  e. Maintenance — ML projects evolve; documented code is far
     easier to update, debug, or retrain on new data months later.
